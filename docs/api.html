🌍 Osmium API Documentation

ARTCC Management & Training Platform

Welcome to the Osmium API

Osmium is a comprehensive backend API for managing ARTCC (Air Route Traffic Control Center) operations, training programs, events, and controller statistics.

This documentation provides complete information about all available endpoints, authentication methods, and response formats.

Authentication Endpoints Error Handling Rate Limits
Base URL: /api/v1

🔐 Authentication

Session-Based Authentication

The Osmium API uses session-based authentication. Users authenticate via VATSIM OAuth2 and receive a session cookie that must be included in subsequent requests.

Getting Your Session

1. Redirect to /api/v1/auth/vatsim/login
2. User completes VATSIM OAuth flow
3. Redirect back to your application with session established
4. Include cookies in all subsequent API requests

User Information

To get current user info and permissions, use: GET /api/v1/me 

curl -X GET http://localhost:3000/api/v1/me \
  -H "Cookie: session=..."

⚠️ Error Handling

The API returns standard HTTP status codes and JSON error responses.

Common Status Codes

200 OK - Request successful
201 Created - Resource created successfully
400 Bad Request - Invalid request format or parameters
401 Unauthorized - Missing or invalid authentication
403 Forbidden - Insufficient permissions
404 Not Found - Resource not found
500 Internal Server Error - Server error

Error Response Format

{
  "error": "unauthorized",
  "message": "User must be authenticated to access this resource"
}

⏱️ Rate Limiting

The API currently does not implement rate limiting, but may in the future. Clients should implement reasonable backoff strategies when receiving 5xx errors.
